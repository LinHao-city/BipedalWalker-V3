# 神经网络架构设计

## 🎯 概述

本项目实现了多种神经网络架构来解决BipedalWalker控制问题。从传统的前馈网络到先进的Transformer模型，每种架构都有其独特的设计理念和适用场景。

## 🏗️ 架构类型

### 1. 前馈神经网络 (FeedForward Network)

**设计理念：**
- 简单直接的映射关系
- 每个时间步独立处理
- 参数共享，计算效率高

**网络结构：**
```
输入 (24维观察)
    ↓
隐藏层1 (256神经元, ReLU + LayerNorm)
    ↓
残差连接
    ↓
隐藏层2 (256神经元, ReLU + LayerNorm)
    ↓
残差连接
    ↓
输出层 (4维动作)
```

**关键特性：**
- ✅ 残差连接缓解梯度消失
- ✅ LayerNorm提高训练稳定性
- ✅ 计算复杂度 O(n)
- ✅ 内存占用小

**适用场景：**
- 马尔可夫决策过程
- 简单控制任务
- 实时性要求高的应用

### 2. 长短期记忆网络 (LSTM)

**设计理念：**
- 利用历史信息改进决策
- 建模时序依赖关系
- 门控机制控制信息流

**网络结构：**
```
历史观察序列 (h步 × 24维)
    ↓
LSTM层 (256隐藏单元)
    ↓
全连接层 (256神经元, ReLU)
    ↓
输出层 (4维动作)
```

**LSTM核心公式：**
```
fₜ = σ(W_f · [hₜ₋₁, xₜ] + b_f)  # 遗忘门
iₜ = σ(W_i · [hₜ₋₁, xₜ] + b_i)  # 输入门
C̃ₜ = tanh(W_C · [hₜ₋₁, xₜ] + b_C)  # 候选值
Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ  # 细胞状态
oₜ = σ(W_o · [hₜ₋₁, xₜ] + b_o)  # 输出门
hₜ = oₜ ⊙ tanh(Cₜ)  # 隐藏状态
```

**关键特性：**
- ✅ 处理长距离时序依赖
- ✅ 门控机制防止梯度消失
- ✅ 选择性记忆和遗忘
- ✅ 适应非马尔可夫环境

**参数配置：**
- 历史长度：6步或12步
- 隐藏单元：256
- 层数：1层LSTM + 1层FC

### 3. 双向LSTM (Bi-LSTM)

**设计理念：**
- 同时利用过去和未来信息
- 提高时序建模能力
- 增强特征表示

**网络结构：**
```
历史观察序列 (h步 × 24维)
    ↓
前向LSTM (256隐藏单元) ←→ 后向LSTM (256隐藏单元)
    ↓
拼接 (512维)
    ↓
全连接层 (256神经元, ReLU)
    ↓
输出层 (4维动作)
```

**关键特性：**
- ✅ 双向信息流
- ✅ 更强的时序建模能力
- ✅ 更丰富的特征表示
- ❌ 计算复杂度翻倍
- ❌ 不适合在线控制

### 4. Transformer

**设计理念：**
- 自注意力机制建模全局依赖
- 并行计算提高效率
- 可扩展性强

**网络结构：**
```
历史观察序列 (h步 × 24维)
    ↓
位置编码
    ↓
Transformer编码器 (N层)
    ├── 多头自注意力
    ├── 残差连接 + LayerNorm
    ├── 前馈网络
    └── 残差连接 + LayerNorm
    ↓
全局平均池化
    ↓
全连接层 (256神经元, ReLU)
    ↓
输出层 (4维动作)
```

**自注意力机制：**
```
Q = XW_Q, K = XW_K, V = XW_V
Attention(Q,K,V) = softmax(QKᵀ/√d_k)V
MultiHead(Q,K,V) = Concat(head₁,...,head_h)Wᵒ
```

**关键参数：**
- 注意力头数：8
- 模型维度：256
- 前馈网络维度：512
- 层数：2-4层
- 位置编码：正弦位置编码

**关键特性：**
- ✅ 建模长距离依赖
- ✅ 并行计算效率高
- ✅ 可解释性强
- ✅ 易于扩展

### 5. 多层感知机 (MLP)

**设计理念：**
- 最基础的神经网络架构
- 简单可靠，易于训练
- 适用于线性可分或近似线性可分问题

**网络结构：**
```
输入 (24维观察)
    ↓
隐藏层1 (512神经元, ReLU)
    ↓
隐藏层2 (256神经元, ReLU)
    ↓
隐藏层3 (128神经元, ReLU)
    ↓
输出层 (4维动作)
```

**关键特性：**
- ✅ 结构简单，易于实现
- ✅ 训练稳定
- ✅ 计算效率高
- ❌ 无法处理时序信息
- ❌ 表达能力有限

## 🔧 技术实现细节

### 激活函数选择

**ReLU (Rectified Linear Unit):**
```python
f(x) = max(0, x)
```
- ✅ 解决梯度消失问题
- ✅ 计算效率高
- ❌ 神经元死亡问题

**GELU (Gaussian Error Linear Unit):**
```python
GELU(x) = x * Φ(x)
```
- ✅ 平滑的激活函数
- ✅ 在Transformer中表现优异
- ❌ 计算复杂度稍高

**Tanh (双曲正切):**
```python
tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)
```
- ✅ 输出范围[-1,1]，适合动作输出
- ✅ 零中心化
- ❌ 梯度消失问题

### 正则化技术

**Layer Normalization:**
```python
μ = 1/H Σᵢ₌₁ᴴ xᵢ
σ² = 1/H Σᵢ₌₁ᴴ (xᵢ - μ)²
ŷ = (xᵢ - μ) / √(σ² + ε) ⊙ γ + β
```

- ✅ 稳定训练过程
- ✅ 允许更高学习率
- ✅ 对批大小不敏感

**Dropout:**
- 训练时随机丢弃神经元
- 防止过拟合
- 提高模型鲁棒性

**权重初始化：**
```python
Xavier初始化: W ~ N(0, 2/(n_in + n_out))
He初始化: W ~ N(0, 2/n_in)
```

### 优化器配置

**AdamW优化器:**
```python
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=4e-4,
    weight_decay=0.0,
    betas=(0.9, 0.999),
    eps=1e-8,
    amsgrad=True
)
```

**学习率调度：**
- 余弦退火学习率衰减
- 梯度裁剪防止梯度爆炸
- 梯度累积处理大批量

3. **复杂度与性能权衡**：Transformer计算复杂但性能稳定
4. **残差连接的重要性**：在前馈网络中提升训练稳定性

## 🎨 架构选择指南

### 什么时候使用前馈网络？

- ✅ **实时性要求高**：推理延迟严格受限
- ✅ **计算资源有限**：嵌入式设备或移动端
- ✅ **环境是马尔可夫的**：当前观察包含所有必要信息
- ✅ **快速原型开发**：需要快速验证想法

### 什么时候使用LSTM？

- ✅ **时序依赖性强**：需要记住历史信息
- ✅ **环境复杂**：非马尔可夫决策过程
- ✅ **探索性任务**：需要长期规划能力
- ✅ **计算资源充足**：可以承担额外开销

### 什么时候使用Transformer？

- ✅ **长距离依赖**：需要建模全局时序关系
- ✅ **可解释性要求高**：需要注意力权重分析
- ✅ **大规模训练**：有足够数据和计算资源
- ✅ **研究目的**：探索最新技术

## 🚀 未来改进方向

### 混合架构
- **LSTM + Transformer**：结合局部和全局建模能力
- **CNN + Transformer**：视觉输入的时序建模
- **分层架构**：低级控制和高级规划分离

### 效率优化
- **知识蒸馏**：大模型向小模型传递知识
- **模型剪枝**：减少冗余参数
- **量化压缩**：降低精度要求

### 自适应架构
- **神经架构搜索**：自动寻找最优架构
- **动态网络**：根据环境复杂度调整结构
- **模块化设计**：可插拔的功能组件
